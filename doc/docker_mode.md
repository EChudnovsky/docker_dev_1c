### Docker Mode
---

В состав образа входит фунционал Docker-mods, который обеспечивает дополнительную функциональность в контейнерах без его пересборки. Подключение модов выполняется во время запуска контейнера.

- [Функционал](#функционал)
- [Пример подключения Docker-mods](#пример-подключения-docker-mods)

## Функционал

Основные возможности функционала Docker-mods:

* этап установки (скачивания) дополнительных пакетов  
* этап инициализации (настройки) mods
* указать зависимости mods друг от другая, что позволяет устанавливать и запускать mods в требуемом порядке
* этап запуска mods как постоянную службу

Подробнее см. [здесь](https://github.com/linuxserver/docker-mods?tab=readme-ov-file)
Список готовых mode см. [здесь](https://mods.linuxserver.io/)

## Пример подключения Docker-mods

Далее приведен пример мода со стандартными инструментами для устранения неполадок (dnsutils, net-tools, iputils-ping, traceroute), которые устанавливаются/обновляются при запуске контейнера. При запуске контейнера Docker необходимо установить переменную среду DOCKER_MODS=linuxserver/mods:universal-tshoot, где значение переменной - это имя репозитория и ветки заранее опубликованных инструментов. Если добавляете несколько модов, их необходимо ввести в виде массива, разделенные |, 
например DOCKER_MODS=linuxserver/mods:universal-tshoot|linuxserver/mods:universal-mod2

```bash

  docker run --name=rdesktop \
    -e DOCKER_MODS=lscr.io/linuxserver/mods:universal-tshoot \
    docker_dev_1c /bin/bash

```
